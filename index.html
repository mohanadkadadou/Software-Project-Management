Chapter 1
Introducing Continuous Integration
المصطلحات الأساسية · الممارسات السبع · قيمة CI

📌 Definition

Continuous Integration (CI) is a software development practice where members of a team integrate their work frequently — usually each person integrates at least daily — which results in multiple integrations per day.

الـ CI (التكامل المستمر) هي ممارسة تطوير حيث يدمج أعضاء الفريق عملهم بشكل متكرر — عادة مرة واحدة على الأقل يوميا — مما يؤدي إلى عمليات تكامل متعددة يوميا.

✨ The 7 CI Practices

  1. Commit Code Frequently
  2. Don't Commit Broken Code
  3. Fix Broken Builds Immediately
  4. Write Automated Developer Tests
  5. All Tests and Inspections Must Pass
  6. Run Private Builds
  7. Avoid Getting Broken Code
Chapter 2
Reducing Risks Using CI
4 مخاطر رئيسية · سيناريوهات حقيقية · حلول · أدوات

⚠️ The 4 Major Risks

1. Lack of Deployable Software

Solution: Use separate integration machine (CruiseControl), put everything in VCS, use Ant/NAnt/Rake

2. Late Discovery of Defects

Solution: Write JUnit tests, run all tests every build, add code coverage tools

3. Lack of Project Visibility

Solution: Configure CruiseControl email/SMS, use Doxygen for auto UML

4. Low-Quality Software

Solution: Use Checkstyle, PMD, JDepend, Simian for code quality

أنواع الاختبارات · التطبيق · المبادلات

✅ Types of Tests

Unit Tests

Tests individual methods/functions in isolation. Written by developers. Fast, run frequently.

Component Tests

Tests a component with dependencies stubbed. Bridges gap between unit and integration.

System/Integration Tests

Tests the full system with all dependencies. Slowest but most comprehensive.
